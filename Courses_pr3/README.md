# Практика №3. Веб-служба учёта лабораторных работ

Демонстрационная веб-служба на чистом Node.js: настройки из трёх источников, ранняя валидация, CORS, rate limiting и защитные заголовки. Предметная область — лабораторные работы (`title`, `maxScore`).

## Запуск

```bash
npm start                                                # учебный режим, порт 3000
npm run start:prod                                       # боевой режим (через APP_MODE)
npm run start:custom                                     # свои лимиты через CLI
node src/server.js --mode=боевой --port=4000             # ручной запуск
```

## Тесты

```bash
npm test                # модульные тесты (node --test)
npm run test:load       # нагрузочный тест (требует запущенного сервера)
```

## API

| Метод  | Путь                       | Описание                            |
|--------|----------------------------|-------------------------------------|
| GET    | `/api/labs`                | список лабораторных работ           |
| GET    | `/api/labs/by-id/:id`      | получить работу по ID               |
| POST   | `/api/labs`                | создать работу (`title`, `maxScore`)|
| GET    | `/api/mode`                | текущий режим и лимиты              |

## Настройки

Источники в порядке возрастания приоритета: файл `config/appsettings.json` → переменные окружения с префиксом `APP_` → аргументы командной строки `--ключ=значение`.

Поддерживаемые ключи: `mode`, `port`, `trustedOrigins`, `rateLimits.readPerMinute`, `rateLimits.writePerMinute`.

## Структура

- `src/config.js` — чтение и валидация настроек
- `src/security.js` — CORS, rate limiter, защитные заголовки
- `src/labs.js` — репозиторий лабораторных работ
- `src/server.js` — HTTP-сервер и маршрутизация
- `test/` — модульные и нагрузочный тесты
